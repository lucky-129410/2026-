回溯算法本质上是在一棵**隐式的决策树**（状态空间树）上进行**深度优先搜索（DFS）**。

我们可以根据“**我们要从原集合中选出什么**”将回溯分为三大类：**子集型**、**组合型**、**排列型**。理解它们的关键在于掌握：**搜索树是怎么长的？每一层在选什么？如何避免重复？**

---

### 1. 子集型回溯 (Subset)
**核心思想：** “选或不选”。对于每一个元素，只有两种状态：**在子集里** 或 **不在子集里**。

*   **搜索方向**：按顺序处理原数组中的每一个元素。
*   **状态转换**：第 $i$ 个元素决定选不选 $\to$ 去看第 $i+1$ 个元素。
*   **结果位置**：搜索树的**每一个节点**都是一个合法的子集。

**Python 模板：**
```python
def subsets(nums):
    ans = []
    path = []
    n = len(nums)
    
    def dfs(i):
        if i == n:
            ans.append(path[:]) # 复制一份当前路径
            return
        
        # 决策1：不选 nums[i]
        dfs(i + 1)
        
        # 决策2：选 nums[i]
        path.append(nums[i])
        dfs(i + 1)
        path.pop() # 显式回溯

    dfs(0)
    return ans
```

---

### 2. 组合型回溯 (Combination)
**核心思想：** “选 $k$ 个数”。它其实是子集型的一种特殊情况（限制了结果的长度）。为了避免选出重复的组合（例如选了 $[1, 2]$ 就不能再选 $[2, 1]$），我们通常规定：**只能从当前位置往后选**。

*   **搜索方向**：通过一个 `start` 索引限制可选范围。
*   **状态转换**：在 `[start, n]` 范围内选一个数 $\to$ 下一层从 `selected_index + 1` 开始选。
*   **结果位置**：只有当 `len(path) == k` 时才记录结果（叶子节点）。

**Python 模板：**
```python
def combine(n, k):
    ans = []
    path = []
    
    def dfs(start):
        # 剪枝：如果剩下的数不够凑成 k 个，直接返回
        if len(path) == k:
            ans.append(path[:])
            return
        
        # 只能从 start 开始往后选，保证了结果是有序的，从而去重
        for i in range(start, n + 1):
            path.append(i)
            dfs(i + 1) # 下一层从 i+1 开始
            path.pop()

    dfs(1)
    return ans
```

---

### 3. 排列型回溯 (Permutation)
**核心思想：** “全员参与，顺序相关”。$[1, 2]$ 和 $[2, 1]$ 是不同的。每一层都要从**还没用过**的数中选一个。

*   **搜索方向**：每一层都尝试所有的数（从 $0$ 到 $n-1$）。
*   **状态转换**：记录哪些数用过了（使用 `used` 数组或你图中看到的 `set` 减法）。
*   **结果位置**：当 `len(path) == n` 时记录结果。

**Python 模板（使用显式回溯）：**
```python
def permute(nums):
    ans = []
    path = []
    used = [False] * len(nums)
    
    def dfs():
        if len(path) == len(nums):
            ans.append(path[:])
            return
        
        for i in range(len(nums)):
            if not used[i]:
                used[i] = True   # 标记使用
                path.append(nums[i])
                dfs()
                path.pop()       # 回溯
                used[i] = False  # 标记释放

    dfs()
    return ans
```

**隐式回溯**
```python

def permute(nums):
    ans = []
    path = [0] * len(nums) # 预先分配空间

    def dfs(i, s):
        # i: 当前要填入的位置下标
        # s: 当前还剩下的、可供选择的数字集合
        if i == len(nums):
            ans.append(path[:])
            return
        
        # 遍历当前剩余可用的数字
        for x in s:
            path[i] = x
            # s - {x} 会产生一个新的 set 对象传给下一层
            # 当前层的 s 并没有变，所以不需要手动 remove 或 add
            dfs(i + 1, s - {x})

    dfs(0, set(nums))
    return ans
```

### 三者核心差异总结

| 类型 | 题目特征 | 关键限制条件 (如何避免重复) | 搜索树形状 |
| :--- | :--- | :--- | :--- |
| **子集型** | 选出所有可能的集合 | 每一个数只有“选/不选”两个分支 | 二叉树 (高度为 $n$) |
| **组合型** | 从 $n$ 个里选 $k$ 个 | **`start` 索引**：只能向后看，不能回头 | 多叉树，随着深度增加宽度变窄 |
| **排列型** | $n$ 个数的全排列 | **`used` 标记**：除了自己，谁都能选 | 多叉树，每一层都在选没用过的数 |

### 如何快速判断用哪种？

1.  **如果顺序不重要**（$[1,2]$ 和 $[2,1]$ 算一个）：
    *   如果是求所有可能，用**子集型**。
    *   如果是求固定长度，用**组合型**（带 `start` 索引）。
2.  **如果顺序很重要**（$[1,2]$ 和 $[2,1]$ 算两个）：
    *   用**排列型**（带 `used` 标记）。

**回到你上一张图的代码：**
它用的是 `s - {x}`，这本质上是**排列型回溯**。它通过集合减法，把“还没用过的数字”传给下一层，从而省去了显式的 `used` 数组和回溯步骤。这种写法非常高级且简洁。
